# Module 12 — Compliance, Audit & Governance

Compliance as an engineering discipline. CIS / NIST CSF / ISO 27001 / PCI DSS / SOC 2 / HIPAA; how each maps to cloud controls and how to *automate the evidence* so an audit costs a quarter, not a quarter-of-the-year-of-engineers.

## Learning objectives

- Map a compliance requirement to a *concrete* cloud control per provider.
- Generate audit evidence programmatically (config rules, posture, query results).
- Implement guardrails-as-code for major frameworks in all three clouds.
- Distinguish "policy" (desire) from "guardrail" (enforced) from "control" (verified).
- Build the bridge between DevOps autonomy and auditor requirements.
