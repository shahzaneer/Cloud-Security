# Module 02 — Identity & Access Management

IAM is where ~70% of cloud breaches happen. This module goes deep on identities, authn flows, federation, the privilege arms race (Red), and least-privilege enforcement at scale (Blue). It is more Red-tilted than most modules — because IAM is the main offensive lever.

## Learning objectives

- Model every identity type per cloud and when each is appropriate.
- Enumerate credential types, rotation, and exposure-path "leakage paths".
- Build assume-role/trust chains and recognize when they become attack graphs.
- Implement least privilege via policy-as-code and just-in-time access.
- Detect identity-based attacks in cloud audit logs.
